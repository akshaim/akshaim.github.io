Function Blocks

Introduction to Function blocks and H/W interfaces.

Function blocks are similar to a class in Object Oriented Programming. Class help us bring abstraction to encapsulate a functionality and promote reuse of functionality, the same applies to function blocks.The following figure represents a function block



A function block in IEC61499 is a passive element. It waits for a event to trigger its functionality.The event and data flow in a function block is shown below

Blinking an LED

We will now do a sample application to blink an LED connected to a GPIO in Raspberry Pi. The plan is to build an application in 4DIAC and then download it to Forte RTE running on RPi. To proceed further we need the following.

  1. PC running 4DIAC
  2. Raspberry Pi with Forte
  3. LED
  4. 220E Resistor (or equivalent)
  5. Breadboard
  6. Some wires/Jumpers

Building Application

  • Launch 4DIAC and select your workspace
  • Create a new system and an application as shown below

  • We need to implement a logic to toggle states.We will need
    1. A flip Flop
    2. Event Generator
  • Select the function blocks as shown below

  • The E_SR Flip Flop requires two event inputs but E_CYCLE generates only one event output. Hence we will need a switch.The selection and connections are shown below

  • The E_CYCLE generate events in an interval provided by DT hence the value needs to be assigned.

    • Quick Tip: Move the mouse over to event or data I/O to more about each.
  • Now we will need to interface a hardware for which we will use QX (Output) block.

    • PARAMS in QX block take the Output Pin (GPIO) Number.Forte uses SysFS which again uses BCM Pin Mapping. You learn about the Pinout from this image or using the command in RPi. 
    gpio readall
  • Now system configuration should be defined.

    • The target machine(Embedded resource) is Raspberry Pi and is connected via ethernet (RJ45/WiFi).Make sure that a local IP address is assigned to Pi for downloading application.Global IP addresses will only work with port forwarding/DDNS.
  • Map the hardware to the Embedded Resource

  • Configure the embedded resource.

  • Ensure that RPi Device is defined as a HOLOBLOC.


    • Before proceeding any further

    Make sure that the work is saved !!

    Downloading Application to RPi

  • Connect LED to GPIO3 (Pin 5) of RPi as shown below


  • Gain terminal access on Pi or use SSH to login remotely.
    More details on Secure Shell (SSH) can be found in my previous RPi tutorial
  • Start FORTE service as root.

  • Launch Deployment Console in 4DIAC and Download the application


  • The LED connected to RPi should now start Blinking.
